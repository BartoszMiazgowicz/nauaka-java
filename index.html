Kolekcje
ArrayList — dynamiczne listy

W przeciwieństwie do zwykłej tablicy int[], ArrayList sama się powiększa gdy dodajesz elementy. Nie musisz znać rozmiaru z góry.

📋

ArrayList<Integer>

Lista liczb całkowitych bez stałego rozmiaru. Dodajesz przez add(), odczytujesz przez get(i), rozmiar przez size().

🗂️

Zagnieżdżona ArrayList (2D)

ArrayList<ArrayList<Integer>> — lista list. Każdy wiersz to osobna lista. Idealnie gdy nie znasz z góry ani liczby wierszy, ani kolumn.

🔢

List<Double> jako wynik

Metoda może zwracać List<Double> — pusta lista oznacza brak wyników, lista z jednym elementem oznacza dokładnie jeden wynik.

Operacje na plikach tekstowych

Java oferuje kilka sposobów na czytanie i zapis plików. Scanner jest najprostszy do czytania wiersz po wierszu — PrintWriter do zapisu.

Czytanie pliku — Scanner
import java.util.Scanner;
import java.io.*;

try (Scanner sc = new Scanner(new File(filename))) {
  while (sc.hasNextLine()) {
    String linia = sc.nextLine().trim();
    if (linia.isEmpty()) continue;
    // podziel po spacji
    String[] czesci = linia.split("\\s+");
    // przetwórz każdą część...
  }
} catch (FileNotFoundException e) {
  System.out.println("Nie znaleziono pliku!");
}
// try-with-resources — Scanner zamknie się sam
Zapis do pliku — PrintWriter
import java.io.*;

try (PrintWriter pw = new PrintWriter(
    new FileWriter("wynik.txt"))) {

  pw.println("Wiersz 1");
  pw.println("Wiersz 2");
  // println → nowa linia po każdym
  // print → bez nowej linii

} catch (IOException e) {
  throw new RuntimeException(e);
}
// plik zamknie się automatycznie
Parsowanie liczb ze Stringa
String linia = "0 1 1 0";
String[] czesci = linia.split("\\s+");

ArrayList<Integer> row = new ArrayList<>();
for (String s : czesci) {
  row.add(Integer.parseInt(s));

// row = [0, 1, 1, 0]

// \\s+ = jedna lub więcej spacji/tabulatorów

Testowanie
JUnit 5 — testy jednostkowe

Test jednostkowy sprawdza jedną konkretną rzecz: czy metoda dla danych wejściowych zwraca oczekiwany wynik. JUnit automatyzuje to sprawdzanie.

🔬

@Test

Adnotacja nad metodą — mówi JUnit "to jest test". Metoda musi być void, bez parametrów, nazwa opisuje co sprawdza.

✅

Asercje

assertEquals(oczekiwana, rzeczywista) — sprawdza równość. assertTrue(warunek) — sprawdza czy warunek jest spełniony.

📦

build.gradle

Potrzebujesz testImplementation z JUnit Jupiter i useJUnitPlatform() w bloku test. Testy lądują w src/test/java.

build.gradle — konfiguracja JUnit 5
plugins { id 'java' }
repositories { mavenCentral() }

dependencies {
  testImplementation
    'org.junit.jupiter:junit-jupiter:5.10.0'
}

test {
  useJUnitPlatform() // włącz JUnit 5
}

Testowanie przypadków brzegowych

Dobry test sprawdza nie tylko "normalny" przypadek — powinien też sprawdzić co się dzieje gdy danych brakuje, są zerowe, albo problem jest nieokreślony.

Przykład: metoda obliczająca pierwiastki równania ax² + bx + c = 0

Przypadek Warunek Oczekiwany wynik Przykład (a, b, c)

Jeden pierwiasteka≠0, Δ = 0lista 1 elementu(1, -4, 4) → 2

Najczęstsze pytania
FAQ

Odpowiedzi na pytania które pojawiają się podczas pracy z plikami i testami.

Jaka różnica między Scanner(File) a Scanner(System.in)?▾
Scanner(new File("nazwa.txt")) czyta z pliku na dysku. Scanner(System.in) czyta z klawiatury (standardowe wejście). Działają tak samo — nextLine(), nextInt() itd. — tylko źródło danych jest inne.
Co to jest try-with-resources?▾
Gdy otwierasz plik przez try (Scanner sc = new Scanner(...)), Java automatycznie zamknie sc po wyjściu z bloku try — nawet jeśli wystąpił wyjątek. Bez tego musisz ręcznie wywołać sc.close() w bloku finally. Try-with-resources jest czystsze i bezpieczniejsze.
Dlaczego w JUnit oczekiwana wartość jest pierwszym argumentem?▾
To konwencja JUnit: assertEquals(OCZEKIWANA, RZECZYWISTA). Gdy test padnie, JUnit wyświetla "expected 2 but was 3" — właśnie dlatego że oczekiwana jest pierwsza. Odwrotna kolejność da mylące komunikaty o błędach.
Gdzie w projekcie Gradle mają leżeć klasy testowe?▾
Kod testowy trafia do src/test/java/... (nie src/main/java). Gradle kompiluje je osobno i uruchamia przez gradlew test. Raport HTML z wynikami testów znajdziesz w build/reports/tests/test/index.html.
Jak sprawdzić czy lista zawiera konkretną wartość w JUnit?▾
Używasz assertTrue(lista.contains(wartość)). Alternatywnie możesz sprawdzić dokładną kolejność: assertEquals(wartość, lista.get(0)). Jeśli kolejność nie jest gwarantowana (np. dwa pierwiastki mogą być w dowolnej kolejności), lepiej użyć contains().
Co zwrócić gdy problem ma nieskończoną liczbę rozwiązań?▾
Jedną z konwencji jest zwrócenie listy z elementem Double.POSITIVE_INFINITY. Dzięki temu metoda zawsze zwraca List<Double> — nie trzeba tworzyć osobnego wyjątku ani zwracać null. Brak rozwiązań = pusta lista; nieskończenie wiele = lista z ∞.

📄 FeaturesCalculator.java
package pl.edu.twoj.pakiet;
import java.util.*;
import java.io.*;

public class FeaturesCalculator {
  private ArrayList<ArrayList<Integer>> binary_mask;
  private Integer[] mass_center;
  private String filename;

  public FeaturesCalculator(String filename) {
    this.filename = filename;
    this.binary_mask = new ArrayList<>();
    this.mass_center = new Integer[2];
  }

  public void read_data() {
    try (Scanner sc = new Scanner(new File(filename))) {
      while (sc.hasNextLine()) {
        String line = sc.nextLine().trim();
        if (line.isEmpty()) continue;
        ArrayList<Integer> row = new ArrayList<>();
        for (String p : line.split("\\s+"))
          row.add(Integer.parseInt(p));
        binary_mask.add(row);
      }
    } catch (FileNotFoundException e) {
      throw new RuntimeException("Nie znaleziono pliku: " + filename);
    }
  }

  public void calculate_mass_center() {
    long sumX = 0, sumY = 0, count = 0;
    for (int row = 0; row < binary_mask.size(); row++)
      for (int col = 0; col < binary_mask.get(row).size(); col++)
        if (binary_mask.get(row).get(col) != 0) {
          sumX += col; sumY += row; count++;
        }
    if (count > 0) {
      mass_center[0] = (int) Math.round((double) sumX / count);
      mass_center[1] = (int) Math.round((double) sumY / count);
    }
  }

  public void save_results() {
    String out = filename.replace(".txt", "_out.txt");
    try (PrintWriter pw = new PrintWriter(new FileWriter(out))) {
      for (int row = 0; row < binary_mask.size(); row++) {
        StringBuilder sb = new StringBuilder();
        for (int col = 0; col < binary_mask.get(row).size(); col++) {
          if (row == mass_center[1] && col == mass_center[0])
            sb.append("P");
          else
            sb.append(binary_mask.get(row).get(col));
          if (col < binary_mask.get(row).size() - 1) sb.append(" ");
        }
        pw.println(sb);
      }
    } catch (IOException e) { throw new RuntimeException(e); }
  }

  public static void main(String[] args) {
    Scanner in = new Scanner(System.in);
    System.out.print("Podaj nazwę pliku: ");
    String f = in.nextLine();
    try {
      FeaturesCalculator fc = new FeaturesCalculator(f);
      fc.read_data(); fc.calculate_mass_center(); fc.save_results();
      System.out.println("Zapisano: " + f.replace(".txt", "_out.txt"));
    } catch (RuntimeException e) {
      System.out.println("Błąd: " + e.getMessage());
    }
  }
}

package pl.edu.twoj.pakiet;
import java.util.*;

public class RootsCalculator {
  public static List<Double> calculate_roots(double a, double b, double c) {
    List<Double> roots = new ArrayList<>();
    if (a != 0) {
      double d = b * b - 4 * a * c;
      if (d > 0) {
        roots.add((-b - Math.sqrt(d)) / (2 * a));
        roots.add((-b + Math.sqrt(d)) / (2 * a));
      } else if (d == 0) {
        roots.add(-b / (2 * a));
      } // d < 0 → pusta lista
    } else if (b != 0) {
      roots.add(-c / b); // liniowe: x = -c/b
    } else if (c == 0) {
      roots.add(Double.POSITIVE_INFINITY); // 0=0: ∞
    } // a=b=0, c≠0 → pusta lista
    return roots;
  }
}
