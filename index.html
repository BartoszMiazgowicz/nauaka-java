Podstawy
Java w pigułce

Wszystko co musisz wiedzieć żeby zrozumieć kod projektu.

📦

Klasy i obiekty

Klasa to szablon (przepis), obiekt to konkretna rzecz stworzona według tego przepisu. Tworzysz go przez new NazwaKlasy().

⚡

Metody statyczne

Metoda z static nie wymaga tworzenia obiektu. Wywołujesz ją przez Klasa.metoda() — tak jak Math.random().

🔒

Prywatny konstruktor

Jeśli klasa ma private konstruktor, nie można zrobić new. Używasz tylko metod statycznych. To wzorzec utility class.

🛡️

Try / Catch

Blok try próbuje wykonać ryzykowny kod. Jeśli coś się nie uda (np. brak internetu), catch przechwytuje błąd.

📋

Package (pakiet)

Pierwsza linia pliku package pl.edu... mówi Javie gdzie logicznie leży klasa. Musi pasować do struktury folderów.

🔗

Import

Żeby użyć klasy z innej biblioteki (np. Jsoup), musisz ją zaimportować przez import org.jsoup.Jsoup; na górze pliku.

Przykład — prosta klasa z metodami:

Kalkulator.java
public class Kalkulator {

  // metoda statyczna — można wywołać bez new Kalkulator()
  public static int dodaj(int a, int b) {
    return a + b;
  }

  // punkt startowy programu — Java zawsze zaczyna tutaj
  public static void main(String[] args) {
    int wynik = Kalkulator.dodaj(3, 5);
    System.out.println("Wynik: " + wynik); // wypisze: Wynik: 8
  }
}

Najczęstsze pytania
FAQ

Odpowiedzi na pytania które pewnie masz.

Co to jest Gradle Wrapper i po co go uruchamiać?▾
Gradle Wrapper to plik gradlew.bat który sam pobiera i uruchamia Gradle. Dzięki temu każdy może uruchomić projekt bez instalowania Gradle ręcznie. Generujesz go raz przez gradle wrapper i potem wszystko działa przez gradlew.
Jaka różnica między java-library a application?▾
java-library to moduł który dostarcza klasy dla innych modułów — nie ma metody main, nie da się go uruchomić bezpośrednio. application to moduł który ma metodę main() i można go odpalić komendą gradlew run.
Po co prywatny konstruktor w PageDownloadingUtility?▾
Prywatny konstruktor blokuje możliwość napisania new PageDownloadingUtility(). To wzorzec "utility class" — klasa która ma tylko metody statyczne i nie ma sensu tworzyć jej obiektów. Wywołujesz metodę bezpośrednio: PageDownloadingUtility.download(url).
Co robi doFirst w Gradle?▾
doFirst to blok kodu który wykonuje się jako pierwszy, przed właściwym działaniem tasku. W zadaniu dodatkowym modyfikujesz task run tak żeby przed uruchomieniem programu wypisał jakie argumenty dostał.
Dlaczego strona kssk.pwr.edu.pl nie działa?▾
Ta strona ma nieważny certyfikat SSL — Java odmawia połączenia dla bezpieczeństwa. To problem serwera uczelni, nie Twojego kodu. Możesz przetestować aplikację na dowolnej innej stronie np. http://example.com — kod jest poprawny.
Teoria
4 filary programowania obiektowego

Java jest językiem obiektowym — cały kod organizujesz wokół tych czterech zasad.

🔒

Enkapsulacja

Ukrywanie szczegółów klasy. Pola oznaczasz jako private, dostęp dajesz przez metody getX()/setX(). Na zewnątrz widać tylko to co potrzebne.

🧬

Dziedziczenie

Klasa może przejąć pola i metody innej klasy. Słowo kluczowe extends. Unikasz powtarzania kodu — wspólne rzeczy piszesz raz w klasie nadrzędnej.

🎭

Polimorfizm

Ten sam kod działa różnie w zależności od obiektu. Metoda może być nadpisana w podklasie i zachowywać się inaczej — bez zmiany reszty kodu.

🗂️

Abstrakcja

Pokazujesz tylko to co istotne, ukrywasz szczegóły. Interfejsy definiują co coś robi, bez opisywania jak — to ustala klasa która je implementuje.

Debugowanie
Najczęstsze błędy i co oznaczają

Java mówi Ci dokładnie co poszło nie tak — musisz tylko wiedzieć jak czytać te komunikaty.

NullPointerExceptionBardzo częsty

Próbujesz użyć zmiennej która ma wartość null — nie wskazuje na żaden obiekt. Sprawdź czy obiekt na pewno został stworzony przez new zanim go użyjesz.

ArrayIndexOutOfBoundsException

Próbujesz odczytać element tablicy pod indeksem który nie istnieje. W projekcie: jeśli zapomnisz podać --args przy uruchomieniu, args[0] wyrzuci ten błąd.

cannot find symbolKompilacja

Java nie może znaleźć klasy lub metody. Najczęstsze przyczyny: literówka w nazwie, brakujący import, lub klasa jest w złym pakiecie.

BUILD FAILEDGradle

Gradle nie mógł zbudować projektu. Czytaj linię > What went wrong: — tam jest konkretna przyczyna. Najczęściej błąd kompilacji lub brak internetu przy pobieraniu bibliotek.

SSLHandshakeExceptionSieć

Strona którą próbujesz pobrać ma nieważny certyfikat HTTPS. Java odmawia połączenia dla bezpieczeństwa. To nie błąd w Twoim kodzie — użyj http://example.com do testów.

📦 Skąd pochodzi nazwa pl.edu.twoj.pakiet?

Pierwsza linia każdego pliku Java to package — mówi Javie gdzie logicznie leży dana klasa.

Nazwa pakietu musi dokładnie odpowiadać strukturze folderów w których leży plik.

Foldery które stworzyłeś: src/main/java/pl/edu/twoj/pakiet
Nazwa pakietu:            pl.edu.twoj.pakiet

Slash / w folderach = kropka . w nazwie pakietu. To ta sama rzecz.

Możesz użyć dowolnej nazwy — np. pl.edu.pwr.lab4 albo com.mojafirma.app.
Ważne żeby foldery i package w kodzie były identyczne.

Konwencja w Polsce: pl = kraj, edu = edukacja, potem nazwa uczelni/projektu/twoja. W zadaniu prowadzący zaproponował pl.edu.twoj.pakiet — możesz zostawić tę nazwę bez zmian.

C StringHtmlCleaner.java — usuwa tagi HTML

Dostaje tekst pełen znaczników HTML jak <h1>Tytuł</h1> i zwraca tylko czysty tekst: Tytuł. Robi to za pomocą biblioteki jsoup.

html-remover / src / main / java / pl / edu / twoj / pakiet / StringHtmlCleaner.java
package pl.edu.twoj.pakiet;
import org.jsoup.Jsoup;
import org.jsoup.nodes.Document;

public class StringHtmlCleaner {
  public String clean(String textToClean) {
    Document document = Jsoup.parse(textToClean);  // czyta HTML
    return document.text();  // zwraca sam tekst
  }
}

D PageDownloadingUtility.java — pobiera stronę z internetu

Dostaje adres strony (np. http://example.com) i zwraca cały jej kod HTML jako tekst. Konstruktor jest prywatny — to znaczy że tej klasy się nie tworzy przez new, tylko od razu wywołujesz jej metodę: PageDownloadingUtility.download(url).

html-remover / src / main / java / pl / edu / twoj / pakiet / PageDownloadingUtility.java
package pl.edu.twoj.pakiet;
import org.jsoup.Jsoup;
import java.io.IOException;

public class PageDownloadingUtility {
  private PageDownloadingUtility() { }  // nie można robić "new"

  public static String download(String url) {
    try {
      return Jsoup.connect(url).get().html();  // idź pod adres i pobierz HTML
    } catch (IOException e) {
      throw new RuntimeException(e);  // jeśli błąd — zgłoś go dalej
    }
  }
}

E build.gradle w html-removing-application — konfiguracja aplikacji

Mówi Gradle że to aplikacja (ma metodę main). Używa modułu html-remover jako biblioteki. Zadanie dodatkowe: doFirst wypisuje argumenty przed uruchomieniem programu.

plugins { id 'application' }  // to jest aplikacja
repositories { mavenCentral() }

  implementation project(':html-remover')  // użyj modułu obok
}
application {
  mainClass = 'pl.edu.twoj.pakiet.Application'  // tutaj jest main()
}
run {

  }
}

F Application.java — główna klasa, serce programu

To jest punkt startowy całej aplikacji. Java zawsze zaczyna od metody main. Program bierze adres strony z argumentu (tego co wpiszesz po --args=), pobiera stronę, czyści ją z tagów i wypisuje tekst na ekran.

html-removing-application / src / main / java / pl / edu / twoj / pakiet / Application.java
package pl.edu.twoj.pakiet;

public class Application {
  public static void main(String[] args) {
    String url = args[0];  // weź pierwszy argument — adres strony
    String html = PageDownloadingUtility.download(url);  // pobierz stronę
    StringHtmlCleaner cleaner = new StringHtmlCleaner();
    String clean = cleaner.clean(html);  // usuń tagi HTML
    System.out.println(clean);  // wypisz na ekran
  }
}
