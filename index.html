Typy generyczne
Generics — kod wielokrotnego użytku

Generics pozwalają pisać klasy i metody które działają dla dowolnego typu — a kompilator sprawdza poprawność typów już na etapie kompilacji, nie w runtime.

📦

Klasa generyczna

Parametr <T> zastępuje konkretny typ. Kompilator podstawia go przy każdym użyciu — Box<String>, Box<Integer> to różne typy w czasie kompilacji.

🔒

Ograniczenia (bounds)

<T extends Comparable<T>> — T musi implementować Comparable. Dzięki temu możesz wywołać compareTo() wewnątrz metody generycznej.

🃏

Wildcards

? extends Number — cokolwiek co rozszerza Number (tylko odczyt). ? super Integer — cokolwiek co jest nad Integer (zapis).

Prosta klasa generyczna Box<T>
public class Box<T> {
  private T value;

  public Box(T value) { this.value = value; }

  public T get() { return value; }

  public <R> Box<R> map(Function<T,R> fn) {
    return new Box<>(fn.apply(value));
  }
}
// użycie:
var b = new Box<>("hello").map(String::toUpperCase);
Metoda generyczna z ograniczeniem
public static <T extends Comparable<T>>
T max(List<T> list) {
  if (list.isEmpty())
    throw new NoSuchElementException();
  T result = list.get(0);
  for (T item : list)
    if (item.compareTo(result) > 0)
      result = item;
  return result;
}
// działa dla Integer, String, Double...
Programowanie funkcyjne
Lambdy i interfejsy funkcyjne

Lambda to anonimowa implementacja interfejsu funkcyjnego — interfejsu z dokładnie jedną abstrakcyjną metodą. Zamiast anonimowej klasy piszesz zwięzłe wyrażenie.

Główne interfejsy funkcyjne
// Function<T, R> — wejście T, wyjście R
Function<String, Integer> len = String::length;
len.apply("Java"); // → 4

// Predicate<T> — wejście T, wyjście boolean
Predicate<String> empty = String::isEmpty;
Predicate<String> notEmpty = empty.negate();

// Consumer<T> — wejście T, brak wyjścia
Consumer<String> print = System.out::println;

// Supplier<T> — brak wejścia, wyjście T
Supplier<List<String>> newList = ArrayList::new;
Składnia lambdy — skróty
// pełna forma
Comparator<String> c1 =
  (a, b) -> { return a.compareTo(b); };

// ciało jednolinijkowe — bez return i {}
Comparator<String> c2 =
  (a, b) -> a.compareTo(b);

// referencja do metody — najkrótsze
Comparator<String> c3 =
  String::compareTo;

// wszystkie trzy są równoważne
Kompozycja funkcji — andThen, compose
Function<String, String> trim =
  String::trim;
Function<String, String> upper =
  String::toUpperCase;

// trim, potem upper
Function<String, String> clean =
  trim.andThen(upper);

clean.apply(" java "); // → "JAVA"
Predicate — łączenie warunków
Predicate<Integer> positive =
  n -> n > 0;
Predicate<Integer> even =
  n -> n % 2 == 0;

// oba warunki jednocześnie
Predicate<Integer> positiveEven =
  positive.and(even);

positiveEven.test(4); // → true
positiveEven.test(-4); // → false
@FunctionalInterfaceWłasny interfejs funkcyjny

Możesz oznaczyć dowolny interfejs z jedną abstrakcyjną metodą adnotacją @FunctionalInterface. Kompilator zgłosi błąd jeśli dodasz drugą metodę abstrakcyjną — chroni to Twój projekt przed przypadkowym zepsuciem kompatybilności z lambdami.

Przetwarzanie danych
Stream API — potoki operacji

Stream to sekwencja elementów, przez którą przepuszczasz dane przez kolejne operacje. Kod jest deklaratywny — mówisz co chcesz uzyskać, nie jak to liczyć.

🔍

Operacje pośrednie (lazy)

filter, map, sorted, distinct, limit — nie wykonują się od razu, budują tylko potok.

🏁

Operacje terminalne

collect, forEach, count, reduce, findFirst — uruchamiają cały potok i zwracają wynik.

📊

Collectors

Collectors.toList(), joining(", "), groupingBy(f), counting() — zbierają wyniki strumienia do różnych struktur.

filter + map + collect
List<String> names = List.of(
  "Anna", "Bartek", "Ada", "Cezary"
);

List<String> result = names.stream()
  .filter(n -> n.startsWith("A"))
  .map(String::toUpperCase)
  .sorted()
  .collect(Collectors.toList());

// result = ["ADA", "ANNA"]
reduce i groupingBy
List<Integer> nums = List.of(1,2,3,4,5);

// suma przez reduce
int sum = nums.stream()
  .reduce(0, Integer::sum);
// sum = 15

// grupowanie po długości nazwy
Map<Integer, List<String>> byLen =
  names.stream()
  .collect(Collectors.groupingBy(
    String::length));
Obsługa braku wartości
Optional — koniec z null

Optional<T> to kontener który może trzymać wartość albo być pusty. Zamiast zwracać null i ryzykować NPE, zwracasz Optional — a wywołujący jest zmuszony obsłużyć brak wartości.

Optional — typowe wzorce użycia
// tworzenie
Optional<String> a = Optional.of("Java");
Optional<String> b = Optional.empty();
Optional<String> c = Optional.ofNullable(maybeNull);

// bezpieczne przetwarzanie — map nie wywołuje się na empty
Optional<Integer> len = a.map(String::length); // Optional[4]
Optional<Integer> lenB = b.map(String::length); // Optional.empty

// wyciągnięcie wartości
int x = len.orElse(0);                       // 4
int y = lenB.orElse(0);                      // 0
int z = len.orElseThrow(); // rzuca NoSuchElementException gdy empty
Najczęstsze pytania
FAQ

Odpowiedzi na pytania które pojawiają się podczas nauki generics, lambd i Stream API.

Czym różni się map od flatMap w Stream?▾
map zamienia każdy element na inny element: Stream<T> → Stream<R>. flatMap zamienia każdy element na strumień, a potem spłaszcza wszystkie strumienie w jeden: użyteczne gdy np. masz listę list i chcesz uzyskać płaską listę wszystkich elementów.
Kiedy używać ? extends T, a kiedy ? super T?▾
Zasada PECS (Producer Extends, Consumer Super): jeśli kolekcja produkuje elementy (czytasz z niej) — używaj ? extends T. Jeśli konsumuje (piszesz do niej) — używaj ? super T. Jeśli robisz obie rzeczy — użyj konkretnego typu T bez wildcard.
Czy stream można przeiterować dwa razy?▾
Nie — stream można skonsumować tylko raz. Po wywołaniu operacji terminalnej (np. collect, forEach) stream jest zamknięty. Próba ponownego użycia rzuca IllegalStateException. Jeśli potrzebujesz wielokrotnego dostępu, zbierz wyniki do listy.
Co to jest type erasure i dlaczego nie można zrobić new T()?▾
W czasie kompilacji Java usuwa informacje o typie generycznym — to type erasure. W bytecode Box<String> i Box<Integer> to ta sama klasa. Dlatego w runtime nie wiadomo co to T i nie można tworzyć instancji. Obejście: przekaż Class<T> clazz i użyj clazz.getDeclaredConstructor().newInstance().
Jaka różnica między orElse a orElseGet w Optional?▾
orElse(default) zawsze ewaluuje default — nawet gdy Optional ma wartość. orElseGet(() -> expensive()) wywołuje dostawcę tylko gdy Optional jest pusty. Jeśli wartość domyślna jest kosztowna do obliczenia, zawsze preferuj orElseGet.
Czy var to dynamiczne typowanie?▾
Nie — var (od Java 10) to wnioskowanie typów lokalnych. Kompilator sam ustala typ zmiennej na podstawie wyrażenia po prawej stronie. Typ jest nadal statyczny i znany w czasie kompilacji — var to tylko skrót składniowy, nie dynamika jak w Pythonie czy JavaScript.

📄 FeaturesCalculator.java
package pl.edu.twoj.pakiet;
import java.util.*;
import java.io.*;

public class FeaturesCalculator {
  private ArrayList<ArrayList<Integer>> binary_mask;
  private Integer[] mass_center;
  private String filename;

  public FeaturesCalculator(String filename) {
    this.filename = filename;
    this.binary_mask = new ArrayList<>();
    this.mass_center = new Integer[2];
  }

  public void read_data() {
    try (Scanner sc = new Scanner(new File(filename))) {
      while (sc.hasNextLine()) {
        String line = sc.nextLine().trim();
        if (line.isEmpty()) continue;
        ArrayList<Integer> row = new ArrayList<>();
        for (String p : line.split("\\s+"))
          row.add(Integer.parseInt(p));
        binary_mask.add(row);
      }
    } catch (FileNotFoundException e) {
      throw new RuntimeException("Nie znaleziono pliku: " + filename);
    }
  }

  public void calculate_mass_center() {
    long sumX = 0, sumY = 0, count = 0;
    for (int row = 0; row < binary_mask.size(); row++)
      for (int col = 0; col < binary_mask.get(row).size(); col++)
        if (binary_mask.get(row).get(col) != 0) {
          sumX += col; sumY += row; count++;
        }
    if (count > 0) {
      mass_center[0] = (int) Math.round((double) sumX / count);
      mass_center[1] = (int) Math.round((double) sumY / count);
    }
  }

  public void save_results() {
    String out = filename.replace(".txt", "_out.txt");
    try (PrintWriter pw = new PrintWriter(new FileWriter(out))) {
      for (int row = 0; row < binary_mask.size(); row++) {
        StringBuilder sb = new StringBuilder();
        for (int col = 0; col < binary_mask.get(row).size(); col++) {
          if (row == mass_center[1] && col == mass_center[0])
            sb.append("P");
          else
            sb.append(binary_mask.get(row).get(col));
          if (col < binary_mask.get(row).size() - 1) sb.append(" ");
        }
        pw.println(sb);
      }
    } catch (IOException e) { throw new RuntimeException(e); }
  }

  public static void main(String[] args) {
    Scanner in = new Scanner(System.in);
    System.out.print("Podaj nazwę pliku: ");
    String f = in.nextLine();
    try {
      FeaturesCalculator fc = new FeaturesCalculator(f);
      fc.read_data(); fc.calculate_mass_center(); fc.save_results();
      System.out.println("Zapisano: " + f.replace(".txt", "_out.txt"));
    } catch (RuntimeException e) {
      System.out.println("Błąd: " + e.getMessage());
    }
  }
}

package pl.edu.twoj.pakiet;
import java.util.*;

public class RootsCalculator {
  public static List<Double> calculate_roots(double a, double b, double c) {
    List<Double> roots = new ArrayList<>();
    if (a != 0) {
      double d = b * b - 4 * a * c;
      if (d > 0) {
        roots.add((-b - Math.sqrt(d)) / (2 * a));
        roots.add((-b + Math.sqrt(d)) / (2 * a));
      } else if (d == 0) {
        roots.add(-b / (2 * a));
      } // d < 0 → pusta lista
    } else if (b != 0) {
      roots.add(-c / b); // liniowe: x = -c/b
    } else if (c == 0) {
      roots.add(Double.POSITIVE_INFINITY); // 0=0: ∞
    } // a=b=0, c≠0 → pusta lista
    return roots;
  }
}

package pl.edu.twoj.pakiet;
import org.junit.jupiter.api.Test;
import java.util.List;
import static org.junit.jupiter.api.Assertions.*;

class RootsCalculatorTest {

  @Test void testDeltaDodatnia_dwaPierwiastki() {
    List<Double> r = RootsCalculator.calculate_roots(1, -5, -6);
    assertEquals(2, r.size()); assertTrue(r.contains(-1.0)); assertTrue(r.contains(6.0));
  }

  @Test void testDeltaZero_jedenPierwiastek() {
    List<Double> r = RootsCalculator.calculate_roots(1, -4, 4);
    assertEquals(1, r.size()); assertEquals(2.0, r.get(0));
  }

  @Test void testDeltaUjemna_brakPierwiastkow() {
    assertEquals(0, RootsCalculator.calculate_roots(1, 1, 6).size());
  }

  @Test void testLiniowe_jedenPierwiastek() {
    List<Double> r = RootsCalculator.calculate_roots(0, 1, 6);
    assertEquals(1, r.size()); assertEquals(-6.0, r.get(0));
  }

  @Test void testSprzecznosc_brakRozwiazan() {
    assertEquals(0, RootsCalculator.calculate_roots(0, 0, 7).size());
  }

  @Test void testNieskonczone_rozwiazania() {
    List<Double> r = RootsCalculator.calculate_roots(0, 0, 0);
    assertEquals(1, r.size()); assertEquals(Double.POSITIVE_INFINITY, r.get(0));
  }
}
